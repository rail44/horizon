# Plan 04 — 再帰レイアウト描画

Status: ready

## 背景

旧ロードマップ Phase 6 の残り。現在の描画は固定4スロットで、
`SplitAxis` は水平のみ(ワークスペースモードの `j/k` が今は no-op なのは
このため)。ビューア群(roadmap「Later」)と多ペイン運用の前提になる。

## スコープ

- `LayoutNode` を再帰的に描画し、3ペイン以上・縦横の分割軸を支える。
- ワークスペースモードの cursor 移動(hjkl の4方向)と focus・IME・
  kitty パススルーの正しさを維持(既存テスト+スモークが緑のまま)。
- 分割コマンドの軸指定(縦/横)をコマンドモデル経由で追加。

## 完了条件

- 縦横混在の3ペイン以上のレイアウトが描画・操作とも正しく動く
  (focus / cursor / IME / パススルー無傷)。
- `j/k` での cursor 上下移動が実レイアウトで意味を持つ。
- ゲート緑+スモークシナリオ(分割系)更新。

## ドメインセッションに委ねる設計判断

- 描画実装の形(再帰ビュー構築の方式、固定スロットの撤去手順)。
- リサイズハンドルを v1 に含めるか(準備だけか)。
- ペイン数の実用上限と縮退挙動。
