# Plan 02 — エージェント出力 UI の抜本改善

Status: ready

## 背景

エージェントの出力が読みづらい(オーナーの実使用)。場当たりの調整では
なく、既存エージェント UI の調査を先に行い、最適解を設計してから実装す
る2段構えを取る。

## 段階1: 調査

既存エージェント UI の transcript 表現を調査し、docs/research/agent-ui.md
に整理する。対象の目安: Claude Code / Claude Desktop、Cursor、Zed、
Warp、opencode、Gemini CLI、Amp 等。観点:

- ツール呼び出しの表現(折り畳み、進行中表示、結果の要約と展開)
- diff・コードブロック・長大出力の扱い
- ストリーミング中の視覚(reasoning の見せ方を含む)
- 承認 UI、追従スクロールと離脱、ターン境界の視覚化
- 「実証や定説」と「そのプロダクト固有の流儀」の区別

## 段階2: 再設計と実装

調査を材料にオーナーとドメインセッション内で設計を固め、実装する。

## 現状の資産と制約

- transcript は末尾200ブロックのウィンドウ+revision メモ化
  (b541d7c)— セッション長に対する不変コストを**回帰させないこと**。
- theme ロール駆動(ハードコード色禁止)。backlog 7(ユーザー吹き出しの
  青と承認バナーの琥珀が中立色に落ちている)はこの再設計で回収候補。
- markdown 描画は src/agent/view/markdown.rs、承認バナーは
  AgentPaneFocus(フォーカス捕捉の前例)。

## 完了条件

- 段階1: research 文書が上がり、オーナーが設計判断に使える状態。
- 段階2: 設計合意の実装がゲート緑+実機確認で着地。
